# База Роста AI — кабинет «чат + дневник»

Урезанная версия ИИ-кабинета для клиента: **один чат с ИИ-психологом (Елизавета) + дневник состояния**.
Тесты и рекомендации из старой версии убраны (тесты остались только в текстовом виде внутри чата — их ведёт сам ИИ по промпту).

## Архитектура

```
Tilda (страница с cabinet.html) ──► Cloudflare Worker ──► DeepSeek API (чат)
                                          │
                                          └──► D1 (профили, история чата, замеры состояния, дневник)
```

Ключ DeepSeek хранится в секретах Worker'а — в коде страницы его нет.

## Файлы

- `worker/worker.js` — код Cloudflare Worker (чат + дневник + память ИИ)
- `worker/schema.sql` — схема базы D1
- `worker/wrangler.toml` — конфиг деплоя
- `frontend/cabinet.html` — страница кабинета (вставить в Tilda через T123 «HTML-код»)

## Деплой (в папке worker/)

```bash
# 1. Логин в Cloudflare
npx wrangler login

# 2. Создать базу D1, скопировать database_id в wrangler.toml
npx wrangler d1 create baza-rosta-db

# 3. Применить схему
npx wrangler d1 execute baza-rosta-db --remote --file=./schema.sql

# 4. Задеплоить (ключ DeepSeek уже в wrangler.toml)
npx wrangler deploy
```

После деплоя wrangler покажет URL воркера (`https://baza-rosta-ai.<аккаунт>.workers.dev`) —
вписать его в `API_URL` в начале `cabinet.html`.

## Как это работает

- **Чат**: фронт шлёт `POST /chat` с tilda_user_id и сообщением. Воркер собирает контекст
  (системный промпт «База роста AI» + память пользователя + последний замер + последние 20 сообщений),
  зовёт DeepSeek, вырезает из ответа скрытый JSON-блок (`state_update`, `memory_update`,
  `diary_entry`, `recommendations`), сохраняет всё в D1 и отдаёт пользователю чистый текст.
- **Дневник**: `GET /state` — последний замер 6 метрик (настроение, энергия, стресс, уверенность,
  мотивация, удовлетворённость) + общий балл; `GET /state/history` — последние 7 замеров для графика.
- **Память**: цели/проблемы/факты/предпочтения копятся в `users.memory_json` и подмешиваются
  в каждый запрос — ИИ «помнит» пользователя между сессиями.
- **Кеш DeepSeek**: профиль пользователя подмешивается в последнее сообщение, а не в системный
  промпт, чтобы префикс контекста оставался стабильным и кеширование работало (дешевле).

## Авторизация Tilda Members

Фронт берёт пользователя из `window.tilda_member`, либо из URL-параметров
(`?tilda_user_id=...&name=...&email=...`), либо генерит локальный dev-id.
